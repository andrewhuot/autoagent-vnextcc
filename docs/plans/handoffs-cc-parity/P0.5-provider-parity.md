**P0.5b — Tool-schema translator.**
- Create `cli/llm/tool_schema_translator.py`:
  - `to_anthropic(schema) -> dict` — passthrough (canonical shape).
  - `to_openai(schema) -> dict` — emit OpenAI function-calling shape (`{"type": "function", "function": {"name": ..., "parameters": <JSON Schema>}}`).
  - `to_gemini(schema) -> dict` — emit Gemini `Tool(function_declarations=[...])` shape (OpenAPI subset).
- Pure function, table-driven. Every bundled tool (~26 of them in `cli/tools/`) must round-trip into each provider's shape without data loss on required fields.
- Tests: `tests/test_tool_schema_translator.py` — per-tool snapshot tests plus edge cases (nested object schema, enum, default values, description preservation).

**P0.5d — OpenAI streaming upgrade.**

### Critical invariants P0.5 must preserve

- **Snapshot stability.** `tests/test_system_prompt.py` stays byte-for-byte.
- **`complete()` stays on the protocol** — existing callers (tests, `EchoModel`, print-mode) keep working.
- **`create_model_client(model="claude-...")` keeps working** — no signature change.
- **Zero mandatory new dependencies for non-Gemini users.** `google-genai` is optional; import lazily; raise a clear error with install instructions if someone picks `gemini-*` without the extra installed.
- **Tool-use semantics unify.** The orchestrator never sees a raw OpenAI `ChatCompletionMessageToolCall` or a Gemini `FunctionCall` — adapters translate to our canonical `AssistantToolUseBlock`.
- **Thinking is normalized.** One `ThinkingDelta` event regardless of provider. UI's "thinking" panel shows `capabilities.thinking` to decide whether to render it at all.
- **Safety blocks surface.** `finish_reason=SAFETY` is not a silent end_turn — it's a distinct `stop_reason="safety"` that the UI shows.

### If you get stuck

- `google-genai` API drift: pin the version, re-read docs via context7.
- OpenAI `tool_calls` deltas arrive with varying indices and can interleave — use the tool-call `id` as the accumulator key, not the index.
- Gemini's streaming iterator yields `GenerateContentResponse` chunks; `finish_reason` may be on the last chunk only — don't infer completion from absence of more text.
- Prompt-cache expectations differ — Anthropic wants explicit breakpoints, OpenAI caches automatically past 1024 tokens, Gemini wants a cached-content handle. If P0.5f complicates the adapter, ship it as no-op for now and revisit with a dedicated `cache_hint_v2` follow-up.
- Test file explosion: consolidate fake-SDK helpers in `tests/fixtures/fake_provider_sdks.py`.
