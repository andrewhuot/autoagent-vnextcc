# P0.5 — Cross-provider parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement this plan task-by-task. Each implementation task also uses `superpowers:test-driven-development`.

**Goal:** Unify Anthropic / OpenAI / Gemini so streaming, tool use, thinking, prompt cache, and cost tracking work on all three providers — not just Anthropic. Downstream phases (P1 streaming dispatch, P2 compaction threshold, P3 classifier per-provider allowlist, P5 per-tool cost) all read from this layer.

**Architecture:**
- Lift capabilities above any one provider via a frozen `ProviderCapabilities` descriptor mounted on every adapter.
- A pure `tool_schema_translator` module converts canonical Anthropic-shape tool schemas to OpenAI function-call and Gemini OpenAPI-subset shapes; every one of the 14 bundled tools must round-trip without data loss.
- A real Gemini adapter replaces today's stub (`factory.py:79-87`), modelled on `AnthropicClient`'s SDK-factory injection pattern.
- OpenAI adapter rewrites its `stream()` from post-hoc event synthesis into real streaming that consumes `choices[0].delta` and accumulates `tool_calls[].function.arguments` JSON strings per `id` before parsing.
- Pricing table keyed by `(provider, model)` replaces today's `capabilities.input_cost_per_1m` / `output_cost_per_1m` driven math in `cost_calculator`, with an overridable `Settings.providers.pricing_overrides` dict cascaded from P0.
- Cache strategy abstraction: `ModelClient.cache_hint()` method — Anthropic honours breakpoints via existing `compute_cache_blocks`; OpenAI and Gemini no-op.

**Tech stack:** Python 3.11, `anthropic` SDK (already installed), `openai` SDK (already installed), `google-genai` (new optional extra), pydantic v2 (already installed via P0), existing `cli.llm.retries.RetryPolicy`.

- Env reads are today direct `os.environ.get(...)` at lines 68, 74. Factory stays env-aware for back-compat.
- For Gemini: must accept **either** `GEMINI_API_KEY` **or** `GOOGLE_API_KEY`, preferring the former (user explicit beats Google-wide default).

### `cli/llm/providers/openai_client.py`
- 280 lines. Non-streaming today. `complete()` calls `client.chat.completions.create(model=..., max_tokens=..., messages=..., tools=...)` at lines 78-84; `stream()` is a degenerate wrapper that calls `complete()` then feeds the result into `events_from_model_response` (lines 105-110).

- `_translate_response` at lines 180-219 already maps `tool_calls` → `AssistantToolUseBlock` and coerces `usage.prompt_tokens`/`completion_tokens`/`total_tokens` into ints. Keep the non-streaming path as the fallback for callers that `complete()` directly (echo tests, print mode callers).

### `cli/llm/providers/anthropic_client.py`
- 322 lines. Canonical shape: streaming + tool-use + thinking + prompt cache + retry + lazy SDK factory.
- `complete()` delegates to `collect_stream(self.stream(...))` — one code path. New adapters follow the same pattern.
- Translates SDK events via duck-typed `_translate_events` (lines 198-253). Emits `TextDelta`, `ThinkingDelta`, `ToolUseStart`, `ToolUseDelta`, `ToolUseEnd`, `UsageDelta`, `MessageStop`.
- Tool input arrives via `input_json_delta` fragments; uses `_tool_id_for_index` helper (lines 275-289) to bind partial-JSON deltas to the most recent tool id (Anthropic keys deltas by block index, not tool id).
- Applies `compute_cache_blocks` at lines 169-178 **inside** `_open_stream`. P0.5f extracts that call into a `cache_hint()` method that orchestrator drives and adapter interprets.

### `cli/llm/types.py`
- 171 lines. `ModelClient` protocol at lines 101-123 declares **only** `complete()`. No `stream`, no `capabilities`. `TurnMessage`, `AssistantTextBlock`, `AssistantToolUseBlock`, `ModelResponse`, `OrchestratorResult` all live here.
- `ModelResponse.stop_reason` is already a free-form string. Adding `"safety"` needs no schema change, only a new documented value.
- `StreamingModelClient` protocol lives in `cli/llm/streaming.py` (lines 120-134) and carries `stream()`. It is **separate** from `ModelClient`. P0.5a merges the streaming contract into the base protocol and adds `capabilities`; `StreamingModelClient` can stay as a deprecated alias or be dropped in favour of the union.

### `cli/llm/capabilities.py`
- **EXISTS** and its purpose is adjacent but distinct from the new `ProviderCapabilities`: this module owns `ModelCapability` — a per-**model** static table with `context_window`, `max_output_tokens`, `supports_tool_use`, `supports_streaming`, `supports_thinking`, `supports_prompt_cache`, `input_cost_per_1m`, `output_cost_per_1m` (frozen dataclass, lines 22-40). It ships 14 curated model entries and drives `resolve_context_limit` / `resolve_max_output`, plus `cost_calculator` (via `get_capability`).
- **Recommendation — create a new file.** Name the new dataclass module `cli/llm/provider_capabilities.py` and keep `cli/llm/capabilities.py` untouched. The two modules answer different questions:
  - `capabilities.py::ModelCapability` — "what does *this model* support, and what does it cost per token" (per-model, static, table).
  - `provider_capabilities.py::ProviderCapabilities` — "what does *this adapter* support at runtime" (per-adapter, declared on the `ModelClient`, used by orchestrator to branch).
- They will evolve together but have different lifetimes and different lookup shapes (string keyed vs. attribute on adapter). Collapsing them now would force cost lookups to carry an adapter reference, which is a worse ergonomic.
- P0.5e will **move** cost data out of `ModelCapability` into the new `pricing.py` table — at that point `ModelCapability.input_cost_per_1m` / `output_cost_per_1m` become deprecated. Keep them for one release, route `cost_calculator` through the new table.

### `cli/llm/caching.py`
- 94 lines. Pure-function helpers: `CacheInput(system_prompt, tool_schema_text, pinned_memory)` → `list[dict]` of content blocks annotated with `cache_control={"type": "ephemeral"}` when the content exceeds `MIN_CACHEABLE_CHARS` (2048). Anthropic-specific shape (`cache_control` is Anthropic's breakpoint marker).
- Reusable as-is from the new `anthropic_client.cache_hint` — no refactor needed here. OpenAI and Gemini never call it.

### `Settings` providers shape (handoff claim to correct)
- The handoff claims `Settings.providers.anthropic.api_key` (nested). **Actual shape is flat:** `Settings.providers.anthropic_api_key`, `Settings.providers.openai_api_key`, `Settings.providers.google_api_key`, `Settings.providers.gemini_api_key` (`cli/settings/schema.py:131-142`). `env_bridge.py` maps `ANTHROPIC_API_KEY`/`OPENAI_API_KEY`/`GOOGLE_API_KEY`/`GEMINI_API_KEY` onto those flat fields.

**Public API:**

```python
def to_anthropic(schema: dict) -> dict:
    """Canonical shape — passthrough. Validates required keys."""

def to_openai(schema: dict) -> dict:
    """Emit OpenAI function-calling shape:
       {"type": "function", "function": {"name", "description", "parameters"}}."""

def to_gemini(schema: dict) -> dict:
    """Emit Gemini function_declaration shape:
       {"name", "description", "parameters": <OpenAPI-subset JSON Schema>}.

       Strips JSON Schema keywords Gemini rejects:
       - `$schema`, `additionalProperties`, `definitions`, `$ref`, `$defs`
       - `format: date-time` is preserved (Gemini supports it) but other
         non-OpenAPI formats (`uri`, `email`) are dropped.
       - `oneOf` / `anyOf` / `not` are rejected with a clear translator error.
    """
```

Implementation is a recursive pure function over JSON Schema. Keep the per-provider filter as a small rule table; no magic.

**Test skeletons:**

```python

    try:
```

**Key translation concerns:**
- System prompt → `system_instruction=` on the `GenerateContentConfig`, not a message.
- Tools → `types.Tool(function_declarations=[to_gemini(t) for t in tools])`.
- **`automatic_function_calling=AutomaticFunctionCallingConfig(disable=True)`** — we want the orchestrator to dispatch, not the SDK.
- Thinking → set `thinking_config=ThinkingConfig(include_thoughts=True)` when the model id starts with `gemini-2.5-` and `capabilities.thinking` is true. Map `part.thought=True` chunks to `ThinkingDelta`.
- Streaming → `client.models.generate_content_stream(model, contents, config)` yields `GenerateContentResponse` chunks. Each chunk may carry `candidates[0].content.parts[]`, `candidates[0].finish_reason`, `usage_metadata` (on last chunk typically). Do not infer completion from absence of more text; use `finish_reason`.
- `finish_reason=SAFETY` → emit `MessageStop(stop_reason="safety")`. Same mapping for `RECITATION`, `PROHIBITED_CONTENT` — all surface as `"safety"`.
- Function call parts → `ToolUseStart` (start) + `ToolUseEnd` (same chunk carries full `function_call.args`, so no Delta needed; still accumulate for shape parity).
- Messages with tool results → Gemini's `Part.from_function_response(name, response=...)`. `tool_result` blocks in our `TurnMessage.content` get mapped here.
- Retry via existing `cli.llm.retries.retry_call` + `_is_retryable_error` (duck-typed on `status_code`/`status`, reuse the Anthropic predicate — safe because we match class-name tokens, not SDK-specific classes).
- `capabilities` declaration: `streaming=True, native_tool_use=True, parallel_tool_calls=False` (Gemini is sequential by default), `thinking=True` (2.5-pro / 2.5-flash), `prompt_cache=False` (SDK lacks the handle API today — follow-up), `vision=True`, `json_mode=True` (supports `response_mime_type="application/json"`), `max_context_tokens=1_000_000`, `max_output_tokens=8192`.

### Task P0.5d — OpenAI streaming upgrade

**Commit:** `feat(llm): stream OpenAI chat completions with real-time tool deltas`

**Files to modify:**
- `cli/llm/providers/openai_client.py` — rewrite `stream()` to call `chat.completions.create(..., stream=True)` and translate chunks in real time. Keep `complete()` as the non-streaming fallback path (used by `events_from_model_response`-style callers); **also** allow `complete()` to delegate to `collect_stream(self.stream(...))` for consistency with Anthropic — pick the simpler impl (delegation is smaller and matches Anthropic).

- `tests/fixtures/fake_provider_sdks.py` — `FakeOpenAISDK` with `scripted_chunks` + `scripted_turn_events(provider="openai", ...)`.

**Key translation rules — these are the bugs people hit:**
1. `choices[0].delta.content` → `TextDelta(text=delta.content)`.
2. `choices[0].delta.tool_calls` arrives as a list; each item carries `index`, `id` (only on the first chunk for that call), `function.name` (only on the first chunk), and `function.arguments` (incremental JSON **string** fragments).
3. **Accumulate per `id`**, not per `index` — indices can interleave across parallel calls.
4. On first chunk for an `id`: emit `ToolUseStart(id, name)`.
5. On each arguments delta: accumulate into a per-id buffer (do NOT parse partial JSON). Emit `ToolUseDelta(id, input_json=fragment)`.
6. When a **new** tool_call id appears or the stream ends (`finish_reason="tool_calls"` or `"stop"`): close the previous id with `ToolUseEnd(id, name, input=json.loads(accumulated_buffer))`. Catch `JSONDecodeError` → `input={}` (orchestrator's existing behaviour).
7. `choices[0].delta.reasoning_content` (o1/o3/o4) → `ThinkingDelta(text=...)`. Treat `choices[0].delta.thought` the same if it appears in future SDK versions.
8. Usage in streaming responses: OpenAI sends `usage` on the final chunk when `stream_options={"include_usage": True}` is set. Always pass that option. Emit `UsageDelta({"input_tokens": usage.prompt_tokens, "output_tokens": usage.completion_tokens, "cache_read_tokens": usage.prompt_tokens_details.cached_tokens if present, "reasoning_tokens": usage.completion_tokens_details.reasoning_tokens if present})`.
9. Final event: `MessageStop(stop_reason="tool_use" if finish_reason=="tool_calls" else finish_reason or "end_turn")`.

```python
class TokenPrice:
    input_per_m: float
    output_per_m: float
    cache_write_per_m: float = 0.0   # Anthropic prompt-cache write cost
    cache_read_per_m: float = 0.0    # Cache-hit discount rate
    reasoning_per_m: float = 0.0     # o1/o3 reasoning-token cost (usually same as output)

PRICING: dict[tuple[str, str], TokenPrice] = {
    ("anthropic", "claude-sonnet-4-5"): TokenPrice(3.0, 15.0, cache_write_per_m=3.75, cache_read_per_m=0.3),
    ("anthropic", "claude-opus-4-6"):   TokenPrice(15.0, 75.0, cache_write_per_m=18.75, cache_read_per_m=1.5),
    ("anthropic", "claude-haiku-4-5"):  TokenPrice(1.0, 5.0, cache_write_per_m=1.25, cache_read_per_m=0.1),
    ("openai", "gpt-4o"):      TokenPrice(2.5, 10.0, cache_read_per_m=1.25),
    ("openai", "gpt-4o-mini"): TokenPrice(0.15, 0.6, cache_read_per_m=0.075),
    ("openai", "gpt-5"):       TokenPrice(5.0, 20.0, cache_read_per_m=2.5, reasoning_per_m=20.0),
    ("openai", "o3-mini"):     TokenPrice(1.1, 4.4, reasoning_per_m=4.4),
    ("gemini", "gemini-2.5-pro"):    TokenPrice(1.25, 10.0),
    ("gemini", "gemini-2.5-flash"):  TokenPrice(0.3, 2.5),
    ("gemini", "gemini-2.0-flash"):  TokenPrice(0.1, 0.4),
}

def resolve_price(provider: str, model: str, overrides: dict[str, dict[str, float]] | None = None) -> TokenPrice:
    """Look up (provider, model) in PRICING, layer overrides on top, fall back
    to a conservative default and log once for unknown pairs."""

def compute_cost(price: TokenPrice, usage: dict[str, int]) -> float:
    """Pure: (input_tokens, output_tokens, cache_read_tokens, reasoning_tokens)
    × price → USD. Cache-read tokens billed at cache_read_per_m instead of
```
