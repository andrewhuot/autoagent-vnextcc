# Agent Skill Generation Brief

## Mission
Build a module that enables AutoAgent to create, enhance, and evolve the actual skills (tools, sub-agents, playbooks) of the agents it optimizes. AutoAgent goes from "tunes your agent" to "evolves your agent."

## Core Concept
The optimizer's blame map identifies failure patterns and skill gaps. This module turns those gaps into concrete, reviewable skill proposals — generated code (ADK) or config (CX) that fills the gap.

## What to Build

### 1. Gap Analyzer (`agent_skills/gap_analyzer.py`)
Analyzes observer reports and blame maps to identify skill gaps:

```python
@dataclass
class SkillGap:
    """An identified gap in the agent's capabilities."""
    gap_id: str
    gap_type: str              # "missing_tool", "missing_sub_agent", "tool_enhancement", 
                                # "missing_playbook_step", "missing_intent", "missing_flow"
    description: str           # "No tool exists for warranty inquiries"
    evidence: list[str]        # Conversation IDs that demonstrate the gap
    failure_family: str        # "tool_error", "routing_error", etc.
    frequency: int             # How many conversations hit this gap
    impact_score: float        # Estimated score improvement if fixed (0-1)
    suggested_name: str        # "check_warranty_status"
    suggested_platform: str    # "adk" or "cx"
    context: dict              # Extracted context from failed conversations
```

The analyzer should:
- Group failures by root cause
- Identify patterns that indicate a missing capability (not just misconfigured existing capability)
- Distinguish between "needs tuning" (existing optimizer handles this) and "needs new skill" (this module)
- Rank gaps by frequency × impact
- Extract context from failed conversations to inform skill generation

```python
@dataclass
class GeneratedSkill:
    """A generated or enhanced agent skill ready for review."""
    skill_id: str
    gap_id: str                    # Links back to the SkillGap
    platform: str                  # "adk" or "cx"
    skill_type: str                # "tool", "sub_agent", "playbook", "intent", "flow"
    name: str
    description: str
    
    # The generated artifact
    source_code: str | None        # For ADK: Python source
    config_yaml: str | None        # For CX: YAML config
    
    # Files to create/modify
    files: list[GeneratedFile]     # List of (path, content, is_new)
    
    # Quality metadata
    eval_criteria: list[dict]      # How to test this skill works
    estimated_improvement: float   # Expected score improvement
    confidence: str                # "high", "medium", "low"
    
    # Review metadata
    status: str                    # "draft", "approved", "rejected", "deployed"
    review_notes: str
    created_at: float

@dataclass
class GeneratedFile:
    """A file to create or modify as part of a generated skill."""
    path: str
    content: str
    is_new: bool                   # True = create, False = patch existing
    diff: str | None               # Unified diff if patching
```

### 4. Skill Validator (`agent_skills/validator.py`)
Validates generated skills before presenting for review:
- **Syntax validation**: Parse generated Python with `ast.parse()`, validate YAML with yaml.safe_load()
- **Type checking**: Verify function signatures have type hints
- **Docstring check**: Ensure tools have proper docstrings
- **Name collision check**: Verify generated names don't conflict with existing skills
- **Import check**: Verify all imports are available
- **Template completeness**: No unfilled Jinja placeholders remain

### 5. Skill Store (`agent_skills/store.py`)
SQLite store for generated skills:
- `save(skill: GeneratedSkill)` — store a generated skill
- `get(skill_id)` — retrieve by ID
- `list(status=None, platform=None)` — filtered list
- `approve(skill_id)` — mark as approved
- `reject(skill_id, reason)` — mark as rejected
- `list_by_gap(gap_id)` — all skills generated for a gap

### 6. CLI Commands
```
autoagent agent-skills analyze                     # Run gap analysis on current state
autoagent agent-skills generate [--gap-id ID]      # Generate skills for identified gaps
autoagent agent-skills list [--status draft|approved|rejected]
autoagent agent-skills show <skill-id>             # Show generated skill with source code
autoagent agent-skills approve <skill-id>          # Approve for deployment
autoagent agent-skills reject <skill-id> [--reason TEXT]
autoagent agent-skills apply <skill-id> --target <path>  # Write approved skill to agent directory
autoagent agent-skills auto [--max-skills 3]       # Full pipeline: analyze → generate → present
```

A page showing the agent's skill evolution:
- **Gaps section**: Cards showing identified gaps with frequency, impact, evidence count
- **Generated skills section**: Code preview with syntax highlighting, approve/reject buttons
- **Timeline**: History of generated and applied skills
- For each generated skill: show the source code in a code block, the gap it addresses, estimated improvement, and a diff preview if it patches existing code

### 9. Integration with Optimizer Loop
Add a hook in the optimization loop that runs gap analysis periodically:
- After every N cycles (configurable), run the gap analyzer
- If high-impact gaps are found, generate skills and present as change cards
- The change card includes: "AutoAgent identified that users frequently ask about warranty status. A new `check_warranty_status` tool has been generated. Review and approve?"
- This uses the existing change card + review system — no new review UX needed
